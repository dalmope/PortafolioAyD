A continuacion se definen y describen los requsitos funcionales del proyecto.

## <a name="Headings"></a>Requerimientos Funcionales


| ID   | Descripción del requisito |
|:--:  | :-------------: |
| RF1  | El sistema debe permitir a los usuarios pacientes registrarse en el aplicativo |
| RF2  | El sistema debe permitir a los usuarios iniciar sesión |
| RF3  | El sistema debe permitir a los usuarios pacientes solicitar una cita médica en el aplicativo |
| RF4  | El sistema debe permitir a los usuarios pacientes recibir notificaciones sobre las citas (fecha, hora, lugar, doctor, asignación o cancelación) |
| RF5  | El sistema debe permitir que los usuarios secretarios puedan listar las solicitudes pendientes por asignación de cita. |
| RF6  | El sistema debe permitir que los usuarios secretarios puedan visualizar el cronograma de citas |
| RF7  | El sistema debe permitir que los usuarios secretarios puedan asignar citas médicas a los pacientes |
| RF8  | El sistema debe permitir que los usuarios secretarios puedan cancelar citas médicas a los pacientes que así lo soliciten |
| RF9  | El sistema debe permitir a los usuarios médicos recibir las historias clínicas de los pacientes para conocer sus antecedentes médicos |
| RF10 | El sistema debe permitir a los usuarios médicos ser notificaciones sobre las citas (fecha, hora, lugar, paciente, asignación o cancelación) |
| RF11 | El sistema debe permitir a los usuarios médicos hacer anexos a la historia clínica de los pacientes para mantener al día sus antecedentes médicos |
| RF12 | El sistema debe permitir al usuario administrador agregar especialidades médicas |
| RF13 | El sistema debe permitir al usuario administrador actualizar especialidades médicas |
| RF14 | El sistema debe permitir al usuario administrador borrar especialidades médicas |
| RF15 | El sistema debe permitir al usuario administrador listar las especialidades médicas. |
| RF16 | El sistema debe permitir al usuario administrador establecer los horarios de atención de los médicos |
| RF17 | El sistema debe permitir al usuario administrador agregar usuarios médicos |
| RF18 | El sistema debe permitir al usuario administrador agregar usuarios pacientes |
| RF19 | El sistema debe permitir al usuario administrador agregar usuarios secretarios |
| RF20 | El sistema debe permitir al usuario administrador actualizar los datos de los usuarios |
| RF21 | El sistema debe permitir al usuario administrador desactivar usuarios |
| RF22 | El sistema debe permitir al usuario administrador listar usuarios |
| RF23 | El sistema debe permitir al usuario administrador listar usuarios por rol |
| RF24 | El sistema debe permitir al usuario administrador generar informes sobre la atención a los pacientes con una periodicidad establecida por el administrador (pacientes atendidos por cada especialidad → fecha, nombre del paciente, quien lo atendió) |
